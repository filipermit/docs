---
title: User not synced error
---

You got this error because you are trying to check permission for a user that is not synced to your app. This can happen if you are trying to access a user that was created in the Auth0 dashboard, but not yet synced into permit.
user key is key sensitive, so make sure you are using the correct key.

### How to fix it

#### Sync the user to permit with Permit web application

You can create this user with our [Users management screen](https://app.permit.io/user-management) of the Permit web application and check there if the user is synced.

#### Sync the user to permit with the API

You can use the API to create the user and sync it to permit. [redoc](https://api.permit.io/v2/redoc#tag/Users/operation/create_user)

#### Sync the user to permit with the SDK

You can use the SDK to create the user and sync it to permit. [Python SDK](/sdk/python/usage-example#create-or-update-a-user-sync-user), [Node SDK](/sdk/nodejs/user/sync-user) and [more](https://docs.permit.io/)
